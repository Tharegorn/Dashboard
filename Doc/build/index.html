    Introduction

    Welcome to the Dashboard API! You can use ou'r API to acces to our dashboard. We have language bindings in Shell and JavaScript! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    This example API documentation page was created with Slate. Feel free to edit it and use it as a base for your own API's documentation.

    Authentication

    Parameter Description
    user The username of you'r account
    password The password

    To login in use this code:

    const axios = require("axios");

axios.post("http://api_endpoint/login", {
  user: "username",
  password: "password",
});

    The above command will send you a payload with token and refresh token if credentials are valids.

    {
  "status": "success",
  "code": "200",
  "token": "jwt_token",
  "refresh_token": "jwt_token"
}

    Make sure to replace register an account before loged in.

    To register in use this code:

    Parameter Description
    user The username of you'r account
    password The password
    confirm_password The password confirmation 
    const axios = require("axios");

axios.post("http://api_endpoint/register", {
  user: "username",
  password: "password",
  confirm_password: "password",
});

    The above command will send you a payload with token and refresh token if credentials are valids.

    {
  "status": "success",
  "code": "200"
}

    Token refreshing

    In order to refresh you'r token this route exists:

    const axios = require("axios");

axios.post("http://api_endpoint/refresh", {
  headers: { Authorization: "Bearer " + refresh_token },
});

    The above command will send you a payload with token and refresh token if credentials are valids.

    You need to provide the Authorization with a bearer and refresh token to authentify.

    {
  "status": "success",
  "code": "200",
  "token": "jwt_token",
  "refresh_token": "jwt_token"
}

    Sticky notes

    Create a new note

    Parameter Description
    content The content of the note to create

    {
  "status": 1,
  "code": "200"
}

    Update a note

    Parameter Description
    content The content of the note to update
    title The title of the note to update
    id The ID of the note to update (required)

    To update a note you need to provide a title or a content

    const axios = require("axios");

axios.post(
  "https://endpoint_url/update_note",
  { title: "title", id: 1 },
  { headers: { Authorization: "Bearer " + token } }
);

  "https://endpoint_url/update_note",
  { content: "content" },
  { headers: { Authorization: "Bearer " + token } }
);

    The above command returns JSON structured like this on success:

    {
  "status": 1,
  "code": "200"
}

    Route access

    In order to know if you hava access to a specific route, you can call the api to know that

    Parameter Description
    route The route name you want to access 
    const axios = require("axios");

axios.get("https://endpoint_url/route?route=/admin", {
  headers: { Authorization: "Bearer " + token },
});

    Response could be like that :

    {
  "status": "Authorized",
  "code:": 200
}

    or if case of missing permissions

    {
  "status": "Unahthorized",
  "code": 401
}

    Youtube

    Channel status

    The channel stats widget is provided in this API, tu use it you only need to call :

    Parameter Description
    channel The channel name you want stats 
    const axios = require("axios");

axios.get("https://endpoint_url/widgets/youtube/channel?channel=channel", {
  headers: { Authorization: "Bearer " + token },
});

    The response will be :

    {
  "status": "success",
  "code": 200,
  "thumbnail": "thumbnail",
  "name": "name",
  "id": "id",
  "views": 10,
  "subs": 1933,
  "vids": 2001,
  "url": "https://youtube.com/channel/id"
}

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
